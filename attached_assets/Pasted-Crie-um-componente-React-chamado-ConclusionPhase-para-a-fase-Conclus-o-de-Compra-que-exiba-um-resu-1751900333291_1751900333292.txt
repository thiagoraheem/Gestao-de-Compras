Crie um componente React chamado ConclusionPhase para a fase "Conclusão de Compra" que exiba um resumo completo e detalhado da solicitação de compra finalizada. O componente deve incluir:

1. Estrutura e Layout:

Use o padrão visual existente do projeto com componentes shadcn/ui
Organize as informações em cards bem estruturados
Implemente um layout responsivo que funcione em desktop e mobile
Adicione um cabeçalho com título "Conclusão da Compra" e número da solicitação
2. Seções Obrigatórias:

A) Resumo da Solicitação:

Número da solicitação, data de criação, status final
Solicitante (nome completo, departamento, e-mail)
Centro de custo e departamento
Categoria da compra e nível de urgência
Justificativa completa
Valor total aprovado vs valor final da compra
B) Fornecedor Selecionado:

Dados completos do fornecedor (nome, contato, e-mail, CNPJ se disponível)
Valor da cotação selecionada
Prazo de entrega prometido vs real
Histórico de desempenho (se disponível)
C) Dados do Pedido de Compra:

Número do pedido de compra gerado
Data de emissão do pedido
Condições de pagamento negociadas
Local de entrega
Status do pedido (entregue, com pendências, etc.)
D) Dados do Recebimento:

Data e hora do recebimento
Responsável pelo recebimento
Status da conferência (conforme/não conforme)
Observações do recebimento
Eventuais pendências ou divergências reportadas
E) Timeline do Processo:

Linha do tempo visual mostrando todas as fases percorridas
Datas de cada aprovação
Tempo total do processo
Aprovadores em cada etapa
F) Itens Recebidos:

Lista detalhada de todos os itens
Quantidade solicitada vs quantidade recebida
Preços unitários e totais
Status de cada item (recebido, pendente, divergente)
3. Funcionalidades Adicionais:

Botão para exportar resumo em PDF
Botão para imprimir o resumo
Opção de enviar resumo por e-mail
Visualizador de anexos relacionados (notas fiscais, comprovantes)
Botão "Arquivar" para mover para fase final
Histórico de todas as comunicações e alterações
4. Aspectos Técnicos:

Utilize os hooks useQuery para buscar todos os dados necessários
Implemente loading states apropriados
Trate casos de erro e dados ausentes
Use formatação adequada para datas, valores monetários e números
Implemente interface TypeScript bem tipada
Siga os padrões de acessibilidade do projeto
5. Estados e Validações:

Mostre badges coloridos para diferentes status
Destaque visualmente itens com pendências
Calcule e exiba métricas de performance (tempo total, economia obtida)
Valide se todos os dados estão completos antes de permitir arquivamento
6. Integração com Backend:

Crie endpoints necessários para buscar dados consolidados
Implemente mutações para arquivamento e exportação
Garanta que todas as informações históricas sejam preservadas
O componente deve ser autossuficiente, providing uma visão 360° completa de todo o processo de compra, servindo como documento final e comprovação de que o processo foi executado corretamente.
